Созданы два приложения 
tcp_signal_server - сервер, принимающий по протоколу TCP (порт 7474, задан 
константой) запросы типа    
from -6 to 0 count 100
(значение запроса - отдать 100 точек для промежутка времени от -6 до 0 секунд 
начиная с текущего момента, т е последние 6 секунд)
и отдающий ответы типа
now <текущее время> data <x1 y1 x2 y2 ... xn yn>
для генерации сигнала используется функция signal_function, имеющая вид
sin(x) * sin(10 * x) * sin(0.1 * x);
(x - параметр, соответствующий unix time stamp (число секунд, прошедшее с
начала эпохи unix)

tcp_client_signal_drawer - приложение, посылающее каждые frameExposeTime=30ms
запросы на сервер и отрисовывающее с помощью qCustomPlot полученный ответ.
Приложение оповещает об успехе или проблеме в соединении с сервером в текстовом
поле.
Есть возможность поменять цвет, количество запрашиваемых и отображаемых точек,
ширину линии и отображаемый временной интервал.
Клиентов можно запускать несколько. При незначительных проблемах с соединением 
изображение может терять плавность вывода.

Направления улучшения (для тестового задания выглядит не очень уместно):
управление частотой обновления (запроса не было поэтому не сделал);
фиксированная дискретизация сигнала (например, сигнал определен в каждой целой
милисекунде, но не определен для дробей);
кеширование результата запроса;
предотвращение повторного вычисления функции (на случай если ее вычисление
окажется дорогим);
улучшение отображения.
